### 6. Command 模式說明

新增 **RelayCommand 與 AsyncRelayCommand** 章節，說明：

#### RelayCommand（同步命令）
- 執行同步方法（`Action`）
- 適用於快速操作（開啟對話框、停止操作）
- 直接執行，會阻塞 UI 執行緒

#### AsyncRelayCommand（非同步命令）
- 執行非同步方法（`Func<Task>`）
- 使用 `async/await`，不會凍結 UI
- 內建 `_isExecuting` 標記，防止重複執行
- 執行期間自動禁用按鈕
- 適用於耗時操作（檔案掃描、資料處理）

**設計目的**：
1. ✅ 解耦 UI 與業務邏輯（MVVM 核心）
2. ✅ 統一管理執行狀態（防止重複點擊）
3. ✅ 提升可測試性（無需 UI 即可測試）
4. ✅ 提高可重用性（一個命令綁定多個 UI 元素）
5. ✅ 增強擴充性（統一添加日誌、錯誤處理、權限檢查）
